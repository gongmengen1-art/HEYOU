## 🆕 核心模式：见脸即得，熟客关系自动沉淀

HEYOU 不再是一套"**先录入、才识别**"的熟客门槛系统。现在的核心流程是：

- **谁来都出片** — 任何出现在镜头里的人脸都会被自动建档并生成手办，零录入、零操作。
- **熟客由系统自己认出来** — 同一个人再次到店，人脸会被自动 **聚类并入同一个人**（一人多张脸、特征越攒越准），无需店员做任何标注。
- **每人每天仅一次** — 以"人"为单位去重，既控成本又制造"今天又来领一张"的稀缺感。
- **仍可退回纯熟客模式** — 只想服务预先录入的熟客，把 `orchestration.auto_enroll` 设为 `false` 即可。

> 这是与旧版**业务逻辑上的本质区别**：从"奖励预先录入的熟客"变成"见脸即得，熟客关系在后台自动沉淀"。

## ✨ 功能亮点

- ✅ **见脸即识别 · 自动聚类到人** — InsightFace（SCRFD 检测 + ArcFace 特征）+ 余弦相似度；**双阈值**判定：足够像并入同一个人、明显不像新建、不确定的跳过，策略"**宁可漏认，不可误认**"；一人多张脸越攒越准
- ✅ **主体一致 · 千人千面** — PuLID + InstantID 锁定五官，**明显是本人**；随机种子保证 **每次造型/细节都不同**，绝不撞图
- ✅ **多姿势模板** — 已支持「站立手办」「盘腿坐」两种造型，均自带展示底座；模板可扩展
- ✅ **每人每天一次** — 以"人"为单位自动去重，既控成本又制造稀缺感（次数可配置，另有全局每日总闸兜底）
- ✅ **老板控制台（深色夜店风）** — 访客/熟客分页管理 / 手动录入 / 生成历史 / 一键重生成 / **手动打印兜底** / 实时状态监控
- ✅ **异步生成不卡机** — 云端单张 ≈ 2.3 分钟，独立 worker 处理，**绝不阻塞摄像头识别**；在途防重、当日防重
- ✅ **边缘端 + 云端架构** — 本地只需普通电脑跑识别与调度，繁重生成放云端，**算力成本灵活可控**
- ✅ **可插拔生成后端** — `mock`（离线自测、不花钱）/ `runninghub`（真生成）一键切换
- ✅ **真打印 · 可裁切贴纸** — 支持 **极印 PixCut S1 裁切打印机**（AI 抠图模切、随主体轮廓裁切贴纸）与普通系统打印机，后端一键切换，**Mac 与 Windows 都能真打印**；含 **调试模式**（走完链路不真打印、不耗材）与连续打印自愈
- ✅ **跨平台一致** — 同一套代码在 **macOS 与 Windows 10** 上功能一致：识别、生成、打印全流程都已真机验证
- ✅ **访客留存自动清理** — 自动录入的访客无活动超过设定天数即清理，避免特征库无限膨胀（手动录入的熟客不受影响）

## 🚀 快速开始

```bash
# 1) 安装依赖（自动创建 .venv，Python 3.11）
uv sync

# 2) 启动演示控制台（同时自动开启人脸识别）
uv run python scripts/run_server.py
#    → 浏览器打开 http://127.0.0.1:8000

# （可选）联网/打印前先做一次自检：识别 / 数据库 / 生成后端
uv run python scripts/smoke_test.py

# （可选）验证「见脸即录 → 自动聚类」的判定逻辑（离线、无摄像头、不花钱）
uv run python scripts/smoke_autoenroll.py
```

启动后：**顾客到店即自动识别、建档、生成并打印（每人每天一次），无需任何预先录入**；在控制台可查看成品、重生成、手动补打，也可手动录入/管理熟客。

> 🪟 **Windows 10**：同一套代码也能在 Windows 跑，功能与 Mac 一致（PixCut 真打印或系统打印机任选，摄像头自动适配）——安装与打印机配置见 **[docs/WINDOWS.md](docs/WINDOWS.md)**。

## ⚙️ 配置

全局配置 `config.yaml`

| 配置项                              | 说明                                                             |
| ----------------------------------- | ---------------------------------------------------------------- |
| `generation.backend`                | `mock`（离线自测、不花钱）｜ `runninghub`（云端真生成）          |
| `generation.runninghub.workflow_id` | 选用的工作流（站立 / 盘腿坐对应不同 ID）                         |
| `orchestration.auto_enroll`         | **见脸即录总开关**（默认 `true`）：任何到店人脸自动建档并聚类到人；设 `false` 回到"仅服务预先录入的熟客"旧模式 |
| `recognition.match_high` · `match_low` | 自动聚类**双阈值**（余弦相似度，越高越严）：`≥ match_high` 判为同一人并入其库、`< match_low` 判为新人、中间区间视为不确定跳过 |
| `recognition.min_face_px` · `enroll_min_det_score` · `enroll_max_pose_deg` | 质量门槛：最小人脸像素 / 最低检测置信度 / 最大俯仰偏航角——想让识别更"松"（多收脸）就放宽这几项 |
| `orchestration.daily_limit`         | 每人每日生成次数上限（默认 `1`）                                 |
| `orchestration.global_daily_cap`    | 全体每日生成总闸，兜底成本（`0` = 不限）                         |
| `storage.history_retention_days`    | 生成历史保留天数（默认 `3`）                                     |
| `storage.visitor_retention_days`    | 自动录入访客无活动超过 N 天清理（默认 `30`；手动录入熟客不受影响） |
| `printing.enabled`                  | 自动打印开关（当前 `false`，以"手动打印"兜底，待硬件定型后开启） |
| `printing.backend`                  | 打印后端：`system`（系统打印机：mac/Linux CUPS、Windows win32print）｜ `pixcut`（驱动官方 App 出可裁切贴纸，**mac + Windows 均支持**） |
| `printing.pixcut.cutout` · `dry_run` · `restart_every` | AI 抠图模切（随轮廓裁切贴纸）/ 调试模式（走完链路不真打印、不耗材）/ 每 N 次重启 App 清画板 |
| `logging.max_bytes` · `backup_count` · `retention_days` | 服务日志单文件轮转大小 · 保留份数 · 启动时清理过期日志天数（默认 `5MB × 5` ≈ 25MB） |

## ❓ 常见问题

**Q：没预先录入的人会怎样？**
A：这正是新版核心——默认**见脸即录**：任何到店的人脸都会自动建档，同一个人多次出现自动聚类为一个人（特征越攒越准、越用越像本人），新面孔自动新建，之后照旧生成并打印。**完全无需预先录入**。若只想服务预先录入的熟客，把 `orchestration.auto_enroll` 设为 `false` 即可恢复"仅熟客触发"。

**Q：为什么每人每天只能一次？**
A：既控制云端成本，又制造"专属稀缺感"、鼓励复访。次数在 `config.yaml` 可调；还可用 `orchestration.global_daily_cap` 设一个全体每日总闸兜底成本。

**Q：生成的形象像不像本人？**
A：InstantID + PuLID 锁定五官，**明显是本人**；同时用随机种子让每次的姿势/细节都不同，**主体一致但绝不重样**。

**Q：会不会认错人 / 把一个人拆成好几份？**
A：自动聚类用**双阈值**（`recognition.match_high` / `match_low`）：足够像才并入已有的人、明显不像才新建、中间不确定的直接跳过，策略"宁可漏认、宁可多建一个，也不误并到别人身上"。想调松紧改 `match_high` / `match_low` 与质量门槛（`min_face_px` 等）。（注意：旧的 `match_threshold` 仅离线 `smoke_test.py` 用，实时识别路径**不读它**。）

**Q：隐私怎么处理？**
A：人脸**特征向量**与肖像存于**本地** SQLite，不出本机；仅在生成时把肖像送至云端 RunningHub 出图。开启"见脸即录"后会自动采集到店人脸，自动录入的访客默认 `storage.visitor_retention_days`（30 天）无活动即清理；实际部署请按当地法规做好告知与留存合规。

**Q：成本大概多少？**
A：本地识别免费；每生成一张消耗 RunningHub 付费额度，且**每人每天封顶一张**（另有全局每日总闸兜底），整体可控。离线自测用 `mock` 后端则 0 成本。

**Q：打印机有要求吗？**
A：支持两种后端，在 `config.yaml` 的 `printing.backend` 切换：`system` 走系统打印机（mac/Linux CUPS、Windows win32print，任意系统打印机）；`pixcut` 驱动 **极印 PixCut S1 裁切打印机**官方 App，支持 **AI 抠图模切**贴纸（随主体轮廓裁切）与铺满打印，**macOS 与 Windows 均可**。需要无耗材验证整条链路时用 `printing.pixcut.dry_run`（走完流程但不真打印）。自动打印（`printing.enabled`）默认关、以手动兜底，现场定型后开启。

> PixCut 后端使用须知（Mac 与 Windows 通用）：通过自动化操作官方极印 App 打印，因此打印时需**保持该 App 已打开并登录**；打印过程会接管屏幕 1~3 分钟，**期间请勿手动操作鼠标键盘**。首次在 macOS 上使用需给运行服务的终端授予「辅助功能 + 屏幕录制」权限；Windows 无需额外权限。逐步安装见 [docs/WINDOWS.md](docs/WINDOWS.md)。

## 💎 商业价值与技术壁垒

**为什么是现在**

- AI 生成能力成熟 + 线下体验经济升级，"刷脸即得专属 IP"首次具备低门槛落地条件
- 门店只需普通电脑 + 摄像头 + 打印机，重算力在云端，**先验证、再投入**

**差异化抓手**

- **专属**（明显是本人）× **稀缺**（每人每天一次）× **可带走**（实体卡片）× **可晒**（社交传播）
- **零门槛体验**：见脸即得，无需扫码、无需注册、无需店员录入

**技术壁垒（不是"套个滤镜"）**

- **身份一致的可控模板化生成**：锁脸（InstantID/PuLID）+ 锁姿势与底座（提示词工程，进阶 ControlNet）+ 受控随机性，做到"千人千面又主体一致"
- **见脸即录的自动身份聚类**：双阈值 + 质量门槛 + 每人多特征库，在无人标注的情况下持续把同一个人越认越准，同时抑制误并与重复建档
- **边云解耦 + 可插拔后端**：换场景只需换模板，算力成本灵活
- **工程化闭环**：识别 → 聚类 → 去重 → 异步生成 → 打印的全链路调度、状态监控与兜底，已跑通真机验证

**可迁移性**

- 同一套引擎可平移到 **餐厅 / livehouse / 展会 / 快闪 / 景区 / 品牌活动**——"见脸即得专属形象"的通用能力

**潜在商业模式**

- 硬件一体机 + 耗材（贴纸）+ SaaS 订阅 / 按张分成；连锁多店统一中台；门店自有 IP 皮肤 / 联名 / 节日限定
