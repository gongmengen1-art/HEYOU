## 🚀 快速开始

```bash
# 1) 安装依赖（自动创建 .venv，Python 3.11）
uv sync

# 2) 启动演示控制台（同时自动开启人脸识别）
uv run python scripts/run_server.py
#    → 浏览器打开 http://127.0.0.1:8000

# （可选）联网/打印前先做一次自检：识别 / 数据库 / 生成后端
uv run python scripts/smoke_test.py
```

启动后：在控制台「录入」页把熟客照片录进去 → 顾客到店即自动识别触发 → 在「熟客管理」页可查看成品、重生成、手动打印。

> 🪟 **Windows 10**：同一套代码也能在 Windows 跑，功能与 Mac 一致（PixCut 真打印或系统打印机任选，摄像头自动适配）——安装与打印机配置见 **[docs/WINDOWS.md](docs/WINDOWS.md)**。

## ⚙️ 配置

全局配置 `config.yaml`

| 配置项                              | 说明                                                             |
| ----------------------------------- | ---------------------------------------------------------------- |
| `generation.backend`                | `mock`（离线自测、不花钱）｜ `runninghub`（云端真生成）          |
| `generation.runninghub.workflow_id` | 选用的工作流（站立 / 盘腿坐对应不同 ID）                         |
| `recognition.match_threshold`       | 余弦相似度阈值，越高越严（宁可漏认，不可误认）                   |
| `orchestration.daily_limit`         | 每人每日生成次数上限（默认 `1`）                                 |
| `storage.history_retention_days`    | 生成历史保留天数（默认 `3`）                                     |
| `printing.enabled`                  | 自动打印开关（当前 `false`，以"手动打印"兜底，待硬件定型后开启） |
| `printing.backend`                  | 打印后端：`system`（跨平台系统打印机：mac/Linux 走 CUPS、Windows 走 win32print）｜ `pixcut`（驱动官方 App 出可裁切贴纸，**mac + Windows 均支持**） |
| `printing.pixcut.cutout`            | PixCut：是否每次 **AI 抠图模切**（贴纸随轮廓裁切；每次消耗一次抠图额度）       |
| `printing.pixcut.dry_run`           | PixCut **调试模式**：走完链路但不点"切割"、不打印、不耗材，日志输出成功       |
| `printing.pixcut.restart_every`     | PixCut：每 N 次打印自动重启 App 清理累积画板（默认 `10`，`0` 关闭）          |
| `logging.max_bytes` · `backup_count`| 服务日志单文件轮转大小 · 保留份数（默认 `5MB × 5` ≈ 25MB 硬上限）             |
| `logging.retention_days`            | 启动时清理超过该天数的旧日志（默认 `7`，`0` 关闭）                            |

## ❓ 常见问题

**Q：陌生人（没预先录入）会怎样？**
A：默认**见脸即录**——任何到店的人脸都会自动建档，同一个人多次出现自动归并为一个人（特征越攒越准），新面孔自动新建，之后照旧生成并打印。若只想服务预先录入的熟客，把 `orchestration.auto_enroll` 设为 `false` 即可恢复"仅熟客触发"。

**Q：为什么每人每天只能一次？**
A：既控制云端成本，又制造"专属稀缺感"、鼓励复访。次数在 `config.yaml` 可调；还可用 `orchestration.global_daily_cap` 设一个全体每日总闸兜底成本。

**Q：生成的形象像不像本人？**
A：InstantID + PuLID 锁定五官，**明显是本人**；同时用随机种子让每次的姿势/细节都不同，**主体一致但绝不重样**。

**Q：会不会认错人？**
A：采用高阈值、"宁可漏认不可误认"策略；可通过 `match_threshold` 调松紧。

**Q：隐私怎么处理？**
A：人脸**特征向量**与肖像存于**本地** SQLite，不出本机；仅在生成时把肖像送至云端 RunningHub 出图。开启"见脸即录"后会自动采集到店人脸，自动录入的访客默认 `storage.visitor_retention_days`（30 天）无活动即清理；实际部署请按当地法规做好告知与留存合规。

**Q：成本大概多少？**
A：本地识别免费；每生成一张消耗 RunningHub 付费额度，且**每人每天封顶一张**，整体可控。离线自测用 `mock` 后端则 0 成本。

**Q：打印机有要求吗？**
A：支持两种后端，在 `config.yaml` 的 `printing.backend` 切换：`system` 走系统打印机（mac/Linux CUPS、Windows win32print，任意系统打印机）；`pixcut` 驱动 **极印 PixCut S1 裁切打印机**官方 App，支持 **AI 抠图模切**贴纸（随主体轮廓裁切）与铺满打印，**macOS 与 Windows 均可**。需要无耗材验证整条链路时用 `printing.pixcut.dry_run`（走完流程但不真打印）。自动打印（`printing.enabled`）默认关、以手动兜底，现场定型后开启。

> PixCut 后端使用须知（Mac 与 Windows 通用）：通过自动化操作官方极印 App 打印，因此打印时需**保持该 App 已打开并登录**；打印过程会接管屏幕 1~3 分钟，**期间请勿手动操作鼠标键盘**。首次在 macOS 上使用需给运行服务的终端授予「辅助功能 + 屏幕录制」权限；Windows 无需额外权限。逐步安装见 [docs/WINDOWS.md](docs/WINDOWS.md)。

## 💎 商业价值与技术壁垒

**为什么是现在**

- AI 生成能力成熟 + 线下体验经济升级，"刷脸即得专属 IP"首次具备低门槛落地条件
- 门店只需普通 Mac + 摄像头 + 打印机，重算力在云端，**先验证、再投入**

**差异化抓手**

- **专属**（明显是本人）× **稀缺**（每日一次）× **可带走**（实体卡片）× **可晒**（社交传播）

**技术壁垒（不是"套个滤镜"）**

- **身份一致的可控模板化生成**：锁脸（InstantID/PuLID）+ 锁姿势与底座（提示词工程，进阶 ControlNet）+ 受控随机性，做到"千人千面又主体一致"
- **边云解耦 + 可插拔后端**：换场景只需换模板，算力成本灵活
- **工程化闭环**：识别→去重→异步生成→打印的全链路调度、状态监控与兜底，已跑通真机验证

**可迁移性**

- 同一套引擎可平移到 **餐厅 / livehouse / 展会 / 快闪 / 景区 / 品牌活动**——"刷脸即得专属形象"的通用能力

**潜在商业模式**

- 硬件一体机 + 耗材（贴纸）+ SaaS 订阅 / 按张分成；连锁多店统一中台；门店自有 IP 皮肤 / 联名 / 节日限定
